Manajemen User Terpusat (LDAP)

Fase terakhir: Membangun sistem direktori untuk menyimpan akun user yang bisa digunakan secara global oleh seluruh server (SSO dasar).

Target Konfigurasi

Struktur Direktori:

  • Base DN: dc=lks2026,dc=id
  • Admin DN: cn=admin,dc=lks2026,dc=id
  • Organizational Unit: People, Groups
  • Server: Internal Server (int-srv)

Kenapa LDAP Penting?

Bayangkan jika ada 100 server, Anda tidak perlu membuat user secara manual di 100 mesin tersebut. Cukup buat di LDAP, mesin lain tinggal "bertanya" ke server ini.

1

Install SLAPD

Software LDAP di Debian.

Lakukan di Internal Server (int-srv):

root@int-srv:~# apt install slapd ldap-utils -y
2

Konfigurasi Domain LDAP

Menyesuaikan dengan domain lks2026.id.

root@int-srv:~# dpkg-reconfigure slapd

*Jangan sampai salah mengetik domain lks2026.id di sini.

3

Membuat Struktur (OU)

Menggunakan file LDIF.

1. Buat file pondasi:

root@int-srv:~# nano struktur.ldif
dn: ou=People,dc=lks2026,dc=id objectClass: organizationalUnit ou: People dn: ou=Groups,dc=lks2026,dc=id objectClass: organizationalUnit ou: Groups

2. Masukkan ke database LDAP:

root@int-srv:~# ldapadd -x -D cn=admin,dc=lks2026,dc=id -W -f struktur.ldif
4

Verifikasi Database

Cek apakah data sudah masuk.

root@int-srv:~# ldapsearch -x -b dc=lks2026,dc=id
