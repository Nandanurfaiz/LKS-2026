| Samba & Internal Server
Storage

Layanan File Server (Samba)

Kita akan membuat server "Gudang Data" di zona Internal agar Client bisa menyimpan dan mengambil file, lengkap dengan autentikasi password.

Target Configuration

VM Info (Internal):

  • Hostname: int-srv
  • IP Address: 192.168.10.20 (Internal)
  • Gateway: 192.168.10.1 (Router fw)
  • Share Name: /data-kantor

Skenario Akses:

  • Read Only: Semua orang bisa baca.
  • Read Write: Hanya user tertentu (dengan password).
  • Akses Client: Via Windows Run (\\IP).
1

Setup VM Internal

Clone & Config IP (VMnet11).

1. Cloning & Adapter

Clone Debian-Master jadi int-srv.

PENTING: Ubah Network Adapter ke Custom: VMnet11 (Internal). Jangan salah pilih!

2. Konfigurasi IP Static

root@int-srv:~# nano /etc/network/interfaces
auto ens33 iface ens33 inet static address 192.168.10.20/24 gateway 192.168.10.1 dns-nameservers 192.168.10.1
root@int-srv:~# hostnamectl set-hostname int-srv
root@int-srv:~# systemctl restart networking
2

Install & Config Samba

Menyiapkan Folder Sharing.

1. Install Paket

root@int-srv:~# apt install samba -y

2. Buat Folder & User

# Buat folder yang mau dishare:
root@int-srv:~# mkdir -p /home/data-kantor
root@int-srv:~# chmod 777 /home/data-kantor

# Buat user khusus Samba:
root@int-srv:~# useradd smbuser
root@int-srv:~# smbpasswd -a smbuser

3. Konfigurasi Share

root@int-srv:~# nano /etc/samba/smb.conf

Tambahkan di baris paling bawah:

[DataKantor] path = /home/data-kantor browsable = yes writable = yes guest ok = no read only = no
root@int-srv:~# systemctl restart smbd
3

Pembuktian (Laptop Fisik)

Strategi CMD (Anti-Gagal).

Cek Koneksi Dulu

Pastikan Laptop bisa "melihat" Server Internal (melewati Router):

C:\> ping 192.168.10.20 Reply from 192.168.10.20: bytes=32 time<1ms TTL=63

Jika RTO: Cek IP Forwarding di Router (`net.ipv4.ip_forward=1`) atau Firewall Windows.

Metode 1: Map Drive via CMD (Sangat Disarankan!)

Ini lebih akurat daripada File Explorer biasa karena menampilkan pesan error jika gagal.

# Format: net use [HurufDrive]: \\[IP_Server]\[NamaShare] /user:[User] [Password] C:\> net use Z: \\192.168.10.20\DataKantor /user:smbuser 1 The command completed successfully.

Jika sukses, buka File Explorer, Drive Z: akan muncul!

Metode 2: Windows Run (Standard)

  • Tekan Windows + R di keyboard.
  • Ketik: \\192.168.10.20 lalu Enter.
  • Jika diminta login, masukkan user: smbuser, password: 1.
  • Buka folder DataKantor.
  • Coba Klik Kanan > New > Text Document (Tes Write Access).

Troubleshooting

Ping RTO ke 192.168.10.20
1. Cek Router (fw): Apakah Routing aktif? `sysctl net.ipv4.ip_forward`.
2. Cek Firewall Windows: Matikan dulu.
3. Cek Gateway: Di Server Internal, `ip route` harus `default via 192.168.10.1`.
"System error 53 has occurred" / "Network path not found"
Ini berarti Laptop tidak bisa menghubungi port Samba. Pastikan IP Server Internal benar dan service samba hidup (`systemctl status smbd`).
