| Samba & NTP Server
Modul B

File Sharing & Time Sync

Membangun Internal Server untuk berbagi file (Samba) ke Client Windows dan menyinkronkan waktu (NTP).

Target Configuration

VM Info (Internal):

  • Hostname: int-srv
  • IP Address: 192.168.10.20 (Internal)
  • Gateway: 192.168.10.1 (Router fw)
  • Folder: /home/siswa/share

Paket Software:

  • Samba: File Sharing untuk Windows.
  • NTP (Sec): Network Time Protocol (Wajib untuk sinkronisasi log).
1

Setup VM Internal

Clone & Config Network.

1. Cloning

Clone Debian-Master menjadi int-srv.

NETWORK ADAPTER: Pilih Custom: VMnet11 (Internal).

2. Konfigurasi IP Static

root@int-srv:~# nano /etc/network/interfaces
auto ens33 iface ens33 inet static address 192.168.10.20/24 gateway 192.168.10.1 dns-nameservers 192.168.10.1
root@int-srv:~# hostnamectl set-hostname int-srv
root@int-srv:~# systemctl restart networking
2

Instalasi & Config Samba

Jantung dari Modul B.

1. Install Paket

root@int-srv:~# apt install samba smbclient -y

2. Buat Folder & User (PENTING!)

User Linux 'siswa' sudah ada dari Master, tapi harus didaftarkan ke Samba.

# 1. Buat folder yang mau dishare
root@int-srv:~# mkdir -p /home/siswa/share
root@int-srv:~# chmod -R 777 /home/siswa/share

# 2. Daftar User ke Samba (Wajib!)
root@int-srv:~# smbpasswd -a siswa

3. Konfigurasi Anti-Gagal (Windows 10/11)

Tanpa setting ini, Laptop Client sering kena error "System Error 64".

root@int-srv:~# nano /etc/samba/smb.conf

A. Bagian [global] (Cari di atas):

workgroup = WORKGROUP # Tambahkan baris sakti ini di bawah workgroup: server min protocol = SMB2 client min protocol = SMB2 client max protocol = SMB3 ntlm auth = yes

B. Bagian Paling Bawah (Definisi Share):

[DataSiswa] path = /home/siswa/share browseable = yes writable = yes guest ok = no read only = no
root@int-srv:~# systemctl restart smbd nmbd
3

NTP Server (Waktu)

Agar jam Client sinkron.

1. Install & Config

root@int-srv:~# apt install ntp -y
root@int-srv:~# nano /etc/ntpsec/ntp.conf

Ubah pool menjadi ID:

pool 0.id.pool.ntp.org iburst

Izinkan Client mengakses (Uncomment/Tambah):

restrict 192.168.10.0 mask 255.255.255.0 nomodify notrap
root@int-srv:~# systemctl restart ntp

Pengujian (Laptop Client)

1. Cek Koneksi (Ping)

ping 192.168.10.20

2. Akses Samba (Cara Paling Stabil)

Gunakan Map Network Drive via CMD untuk melihat error yang jelas:

net use Z: \\192.168.10.20\DataSiswa /user:siswa 1

*Ganti '1' dengan password smbpasswd Anda.

Target: "The command completed successfully". Cek File Explorer drive Z:.

3. Cek NTP

Control Panel > Date and Time > Internet Time > Change Settings. Masukkan IP 192.168.10.20 lalu Update Now.
