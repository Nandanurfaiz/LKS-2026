Layanan Internal & Keamanan

Di fase terakhir ini, kita akan membuat server berbagi file (Samba) dan mengamankan Web Server kita menggunakan sertifikat SSL (HTTPS).

Target Configuration

VM Info (Internal):

  • Hostname: int-srv
  • IP Address: 192.168.10.10 (INT)
  • Gateway: 192.168.10.1 (Router fw)
  • Layanan: Samba File Server

Misi Utama:

  • Membuat folder sharing "DataGuru" & "DataSiswa".
  • Mengamankan Web01/Web02 dengan HTTPS (Self-Signed).
  • Pengujian akses folder dari Laptop Fisik.
1

Setup Server Internal

Membuat VM baru di zona INT.

1. Cloning

Matikan Debian-Master. Clone menjadi int-srv.
Network Adapter: Pilih VMnet11 (INT).

2. Konfigurasi IP

root@int-srv:~# nano /etc/network/interfaces
auto ens33 iface ens33 inet static address 192.168.10.10/24 gateway 192.168.10.1 dns-nameservers 192.168.10.1
root@int-srv:~# hostnamectl set-hostname int-srv
root@int-srv:~# systemctl restart networking

PENTING: Jangan lupa masukkan DVD-1 dan jalankan apt-cdrom add di VM ini.

2

Samba File Server

Berbagi folder dengan Windows.

1. Install Samba

root@int-srv:~# apt install samba -y

2. Buat Folder & User

root@int-srv:~# mkdir -p /home/share/guru
root@int-srv:~# chmod 777 /home/share/guru
root@int-srv:~# useradd guru
root@int-srv:~# smbpasswd -a guru

3. Konfigurasi Smb.conf

root@int-srv:~# nano /etc/samba/smb.conf

Tambahkan di baris paling bawah:

[DataGuru] path = /home/share/guru valid users = guru read only = no browsable = yes
root@int-srv:~# systemctl restart smbd

Pengujian (Laptop Client)

Tes akses folder dari Laptop Fisik Anda:

  • Tekan Win + R di Laptop.
  • Ketik: \\192.168.10.10 lalu Enter.
  • Jika diminta login, masukkan user: guru, password: 123.
  • Coba buat file baru di dalam folder tersebut.
3

Secure Web (HTTPS)

Enkripsi Web Server (Web01).

Pindah ke VM Web01 (172.16.10.10) untuk langkah ini!

1. Buat Sertifikat SSL

Membuat sertifikat "palsu" (Self-Signed) yang berlaku 365 hari:

root@web01:~# mkdir /etc/nginx/ssl
root@web01:~# openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/nginx/ssl/lks.key -out /etc/nginx/ssl/lks.crt

2. Config Nginx SSL

root@web01:~# nano /etc/nginx/sites-available/default

Tambahkan blok server baru di bawah konfigurasi lama (atau edit yang ada):

server { listen 443 ssl; server_name lks2026.id; ssl_certificate /etc/nginx/ssl/lks.crt; ssl_certificate_key /etc/nginx/ssl/lks.key; root /var/www/html; index index.html; }
root@web01:~# systemctl restart nginx

Pengujian (Laptop Client)

  • Buka Browser di Laptop.
  • Ketik: https://172.16.10.10 (Akses langsung IP Web01 lewat Router).
  • Browser akan menampilkan peringatan "Not Secure" (Wajar, karena sertifikat buatan sendiri).
  • Klik Advanced -> Proceed to... (unsafe).
  • Jika web terbuka dengan gembok merah/coret, berarti HTTPS Sukses!
