Fase 3 Public Services
| DNS & Secure Web (Simple Mode)
DMZ Area

Layanan Publik (DMZ)

Fokus utama: Membangun DNS Server dan Web Server yang aman (HTTPS) dengan cara yang cepat dan efisien sesuai standar kisi-kisi.

Target Configuration

Topology Info:

  • Domain: lks2026.id
  • Web01: 172.16.10.10 (HTTPS)
  • Web02: 172.16.10.20 (HTTPS)
  • Gateway: 172.16.10.1 (Router fw)

Misi Utama:

  • Install Apache2 di Web01 & Web02.
  • Aktifkan HTTPS (Port 443) pakai cara cepat (default-ssl).
  • Config DNS Server (Bind9) di Router.
1

Siapkan VM Web Server

Cloning & Network Setup.

1. Cloning VM

Matikan `Debian-Master`. Lakukan Full Clone dua kali:

  • Clone 1 Name: web01
  • Clone 2 Name: web02

Setting Network (Kedua VM): Gunakan 1 Adapter saja -> Custom: VMnet10 (DMZ).

2. Config IP Web01

root@web01:~# nano /etc/network/interfaces
auto ens33 iface ens33 inet static address 172.16.10.10/24 gateway 172.16.10.1 dns-nameservers 8.8.8.8
root@web01:~# hostnamectl set-hostname web01
root@web01:~# systemctl restart networking

3. Config IP Web02

root@web02:~# nano /etc/network/interfaces
auto ens33 iface ens33 inet static address 172.16.10.20/24 gateway 172.16.10.1 dns-nameservers 8.8.8.8
root@web02:~# hostnamectl set-hostname web02
root@web02:~# systemctl restart networking
2

Secure Web (Cara Cepat)

Aktifkan HTTPS dalam 3 perintah saja.

Persiapan DVD (Wajib di Web01 & 02)

# Masukkan DVD-1 di VMware, lalu:
root@web01:~# apt-cdrom add
root@web01:~# apt update

1. Install & Aktifkan SSL

Lakukan perintah "sakti" ini di KEDUA server:

# 1. Install Apache
root@web01:~# apt install apache2 -y # 2. Aktifkan Modul SSL (Agar support HTTPS)
root@web01:~# a2enmod ssl # 3. Aktifkan Config Default SSL (Pakai sertifikat bawaan/snakeoil)
root@web01:~# a2ensite default-ssl # 4. Restart
root@web01:~# systemctl restart apache2
Info: Sertifikat otomatis pakai ssl-cert-snakeoil.pem. Web sudah bisa diakses via https://IP.

2. Ubah Konten (Pembeda)

root@web01:~# echo "<h1>WEB 01 (Secure)</h1>" > /var/www/html/index.html
root@web02:~# echo "<h1>WEB 02 (Secure)</h1>" > /var/www/html/index.html
3

DNS Server (Bind9)

Konfigurasi di Router (fw).

1. Install Bind9

root@fw:~# apt install bind9 dnsutils -y

2. Config Zone

root@fw:~# nano /etc/bind/named.conf.local
zone "lks2026.id" { type master; file "/etc/bind/db.lks"; };

3. Config Database

root@fw:~# cp /etc/bind/db.local /etc/bind/db.lks
root@fw:~# nano /etc/bind/db.lks

Edit menjadi (arahkan ke IP Router 172.16.10.1):

; BIND data file for lks2026.id $TTL 604800 @ IN SOA lks2026.id. root.lks2026.id. ( 2 ; Serial 604800 ; Refresh 86400 ; Retry 2419200 ; Expire 604800 ) ; Negative Cache TTL ; @ IN NS lks2026.id. @ IN A 172.16.10.1 www IN A 172.16.10.1

4. Restart & Cek

root@fw:~# systemctl restart bind9
root@fw:~# nslookup www.lks2026.id localhost

Hasil Akhir Fase 3

Web01 & 02: Apache Aktif (Port 80 & 443).
HTTPS: Aktif (Snakeoil Cert).
DNS: nslookup www.lks2026.id OK.
