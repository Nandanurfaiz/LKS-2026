DMZ Area

Layanan Publik (Mode DVD)

Kita akan membangun Web Server dan DNS menggunakan paket instalasi dari DVD (Offline), serta menggunakan Nginx sebagai Load Balancer.

Target Configuration

Topology Info:

  • Domain: lks2026.id
  • Web01: 172.16.10.10 (VM Baru)
  • Web02: 172.16.10.20 (VM Baru)
  • Gateway: 172.16.10.1 (Router fw)

Misi Utama:

  • Install Nginx di Web01 & Web02 (via DVD).
  • Config DNS Server (Bind9) di Router.
  • Config Load Balancer (Nginx) di Router.
1

Siapkan Web Server (VM)

Cloning & Network Setup.

1. Cloning VM

Matikan `Debian-Master`. Lakukan Full Clone dua kali:

  • Clone 1 Name: web01
  • Clone 2 Name: web02

Setting Network (Kedua VM): Gunakan 1 Adapter saja -> Custom: VMnet10 (DMZ).

2. Config IP Web01

Nyalakan web01, login root, edit network:

root@web01:~# nano /etc/network/interfaces
auto ens33 iface ens33 inet static address 172.16.10.10/24 gateway 172.16.10.1 dns-nameservers 8.8.8.8
root@web01:~# hostnamectl set-hostname web01
root@web01:~# systemctl restart networking

3. Config IP Web02

Lakukan hal sama di web02, tapi IP beda:

auto ens33 iface ens33 inet static address 172.16.10.20/24 gateway 172.16.10.1 dns-nameservers 8.8.8.8
root@web02:~# hostnamectl set-hostname web02
root@web02:~# systemctl restart networking
2

Install Web Server (Mode DVD)

Karena Web01 & 02 adalah VM baru, kita harus setup DVD dulu.

Persiapan DVD (Wajib di Web01 & 02)

Karena ini VM baru, mereka belum kenal DVD. Lakukan ini di KEDUA server:

# Masukkan DVD-1 di VMware, lalu:
root@web01:~# apt-cdrom add
root@web01:~# apt update

# Lakukan hal yang sama untuk DVD-2 jika diperlukan

1. Install Nginx

root@web01:~# apt install nginx -y

*Jika diminta masukkan DVD-2, ganti file ISO di setting VMware lalu tekan Enter.

2. Ubah Tampilan Web01

root@web01:~# echo "

Ini Server WEB 01

" > /var/www/html/index.html

3. Ubah Tampilan Web02

root@web02:~# echo "

Ini Server WEB 02

" > /var/www/html/index.html
3

DNS Server (Bind9)

Konfigurasi di Router (fw).

1. Install Bind9

Pindah ke VM fw (Router). Pastikan DVD terpasang.

root@fw:~# apt install bind9 dnsutils -y

2. Buat Zone Domain

root@fw:~# nano /etc/bind/named.conf.local
zone "lks2026.id" { type master; file "/etc/bind/db.lks"; };

3. Buat File Database

root@fw:~# cp /etc/bind/db.local /etc/bind/db.lks
root@fw:~# nano /etc/bind/db.lks

Edit menjadi:

; BIND data file for lks2026.id $TTL 604800 @ IN SOA lks2026.id. root.lks2026.id. ( 2 ; Serial 604800 ; Refresh 86400 ; Retry 2419200 ; Expire 604800 ) ; Negative Cache TTL ; @ IN NS lks2026.id. @ IN A 172.16.10.1 www IN A 172.16.10.1

4. Restart & Cek

root@fw:~# systemctl restart bind9
root@fw:~# nslookup www.lks2026.id localhost
4

Load Balancer (NGINX)

Kita pakai Nginx karena DVD-1 pasti punya paketnya.

1. Install Nginx di Router

root@fw:~# apt install nginx -y

2. Konfigurasi Load Balancer

Hapus semua isi file default, lalu ganti total.

root@fw:~# nano /etc/nginx/sites-available/default

Hapus semuanya (Ctrl+K), lalu paste:

upstream backend_lks { server 172.16.10.10; server 172.16.10.20; } server { listen 80; server_name lks2026.id www.lks2026.id; location / { proxy_pass http://backend_lks; } }

3. Restart & Test

root@fw:~# systemctl restart nginx
# Cek apakah load balancer jalan (hasil harus berubah-ubah):
root@fw:~# curl 172.16.10.1
root@fw:~# curl 172.16.10.1

Troubleshooting

Error: "Package not found" / "Insert Disc"
Penyebab: Paket ada di DVD-2 tapi yang masuk DVD-1 (atau sebaliknya).
Solusi: Ganti file ISO di VMware Settings, lalu tekan Enter di terminal.
Nginx Gagal Restart (Job failed)
Penyebab: Salah ketik (typo) di file config atau kurang titik koma (;).
Solusi: Cek `nginx -t` untuk melihat baris mana yang error. Pastikan kurung kurawal `{}` lengkap.
