Jantung Jaringan

Kita akan mengubah VM Master menjadi Router (fw) yang menghubungkan Internet, DMZ, Internal, dan Client.

Job Sheet Configuration

Target IP (fw):

  • eth0 (ISP) : DHCP (Bridged)
  • eth1 (DMZ) : 172.16.10.1/24
  • eth2 (INT) : 192.168.10.1/24
  • eth3 (Cli) : 10.10.10.1/24

Tugas Utama:

  • Aktifkan IP Forwarding.
  • Config NAT (Masquerade).
  • Install DHCP Server (Client).
1

Cloning & Hardware

Gandakan Master jadi VM "fw".

1. Full Clone

Klik kanan Debian-Master > Manage > Clone. Pilih Full Clone. Nama VM: fw.

2. Network Adapter

Atur 4 Adapter di VM Settings:

No Setting Fungsi
1 Bridged Internet
2 VMnet10 DMZ
3 VMnet11 Internal
4 VMnet12 Client
2

Konfigurasi IP

Nyalakan VM & Login Root.

1. Cek Interface

Lihat nama asli (ens33/37/etc):

root@fw:~# ip link show

2. Edit File Interfaces

root@fw:~# nano /etc/network/interfaces

Hapus semua, Paste kode ini (Sesuaikan nama 'ens' Anda):

# Loopback auto lo iface lo inet loopback # INTERNET (Bridged) auto ens33 iface ens33 inet dhcp # DMZ (Web Server Gateway) auto ens37 iface ens37 inet static address 172.16.10.1/24 # INTERNAL (Mail/LDAP Gateway) auto ens38 iface ens38 inet static address 192.168.10.1/24 # CLIENT (PC Siswa Gateway) auto ens39 iface ens39 inet static address 10.10.10.1/24

3. Apply Config

root@fw:~# systemctl restart networking
root@fw:~# ip a
3

Routing & NAT

Berbagi Koneksi Internet.

1. IP Forwarding

root@fw:~# nano /etc/sysctl.conf

Hapus pagar (#) di baris:

net.ipv4.ip_forward=1

Aktifkan:

root@fw:~# sysctl -p

2. NAT Masquerade

Install tools & pasang rule:

root@fw:~# apt install iptables iptables-persistent -y

*Pilih YES saat ditanya simpan rule IPv4/IPv6.

# Rule NAT ke ens33 (Internet)
root@fw:~# iptables -t nat -A POSTROUTING -o ens33 -j MASQUERADE
# Simpan Permanen
root@fw:~# netfilter-persistent save
4

DHCP Server

IP Otomatis untuk Client.

1. Install & Set Interface

root@fw:~# apt install isc-dhcp-server -y root@fw:~# nano /etc/default/isc-dhcp-server
INTERFACESv4="ens39"

*Isi dengan interface Client (ens39/ens36).

2. Config Subnet

root@fw:~# nano /etc/dhcp/dhcpd.conf

Scroll paling bawah, tambahkan:

subnet 10.10.10.0 netmask 255.255.255.0 { range 10.10.10.10 10.10.10.100; option routers 10.10.10.1; option domain-name-servers 8.8.8.8; }

3. Start Service

root@fw:~# systemctl restart isc-dhcp-server
root@fw:~# systemctl status isc-dhcp-server

Troubleshooting (Bantuan)

Error: "No DHCPOFFERS received"
Penyebab: VM Bridge tidak nyambung ke WiFi asli.
Solusi: Buka Virtual Network Editor > VMnet0 > Ubah "Automatic" jadi nama WiFi Laptop Anda.
Error: "Job for isc-dhcp-server failed"
Penyebab: Typo di `dhcpd.conf` (kurang titik koma ;) atau salah nama interface di `default/isc-dhcp-server`.
Solusi: Cek ulang ejaan nama interface (ens39 vs ens36) dan tanda kurung { }.
Ping Google Gagal
Penyebab: DNS Server kosong.
Solusi: `nano /etc/resolv.conf`, pastikan ada `nameserver 8.8.8.8`.
