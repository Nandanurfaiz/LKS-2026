Membangun Jantung Jaringan

Kita akan mengubah VM Master menjadi Router (fw) yang menghubungkan Internet, DMZ, Internal, dan Client. Tanpa router ini, topologi hanyalah pulau-pulau yang terpisah.

Job Sheet: Router Configuration

Target IP Address (fw):

  • eth0 (Internet): DHCP (Bridged)
  • eth1 (DMZ) : 172.16.10.1/24
  • eth2 (INT) : 192.168.10.1/24
  • eth3 (Client) : 10.10.10.1/24

Tugas Utama:

  • Mengaktifkan IP Forwarding.
  • Konfigurasi NAT (Masquerade) agar klien bisa internetan.
  • Install DHCP Server untuk melayani Client (Budi).
1

Cloning & Hardware Setup

Menggandakan Master menjadi VM baru bernama "fw".

1. Lakukan Full Clone

Klik kanan Debian-Master > Manage > Clone.
Pilih Create a full clone. Beri nama VM: fw (singkatan Firewall).

2. Tambahkan Network Adapter (PENTING!)

VM fw butuh 4 kaki (interface). Klik Edit virtual machine settings pada VM fw. Atur seperti ini:

Adapter Setting VMware Tujuan
Adapter 1 Bridged (Replicate ON) Internet (ISP)
Adapter 2 Custom: VMnet10 DMZ
Adapter 3 Custom: VMnet11 Internal
Adapter 4 Custom: VMnet12 Client

Jika Adapter kurang, klik tombol Add... > Network Adapter > Finish.

2

Konfigurasi IP Address

Nyalakan VM fw dan login sebagai root.

1. Cek Nama Interface

Nama interface bisa berbeda (ens33/eth0). Cek dulu:

root@fw:~# ip link show

Biasanya urut: ens33 (Bridged), ens37 (DMZ), ens38 (INT), ens39 (Client).
Sesuaikan script di bawah dengan nama yang muncul di layar Anda.

2. Edit Interfaces

root@fw:~# nano /etc/network/interfaces

Hapus isinya, ganti dengan konfigurasi ini (Ingat: Jangan Typo!):

# The loopback network interface auto lo iface lo inet loopback # JALUR INTERNET (Bridged) # Fungsi: Minta IP otomatis dari WiFi/Modem agar router bisa internetan auto ens33 iface ens33 inet dhcp # JALUR DMZ (VMnet10) # Fungsi: IP Statis untuk Gateway Server Web (172.16.10.1) auto ens37 iface ens37 inet static address 172.16.10.1/24 # JALUR INTERNAL (VMnet11) # Fungsi: IP Statis untuk Gateway Server Kantor/Mail (192.168.10.1) auto ens38 iface ens38 inet static address 192.168.10.1/24 # JALUR CLIENT (VMnet12) # Fungsi: IP Statis untuk Gateway Client (10.10.10.1) auto ens39 iface ens39 inet static address 10.10.10.1/24

3. Restart Networking

root@fw:~# systemctl restart networking
root@fw:~# ip a # Pastikan IP sudah muncul semua
3

Routing & NAT (Internet Sharing)

Agar server di DMZ/INT bisa update paket dari internet.

1. Aktifkan IP Forwarding

root@fw:~# nano /etc/sysctl.conf

Hilangkan tanda pagar (#) pada baris:

net.ipv4.ip_forward=1

Simpan, lalu terapkan:

root@fw:~# sysctl -p

2. Konfigurasi NAT (Masquerade)

Kita gunakan iptables. Pastikan paket terinstall:

root@fw:~# apt install iptables iptables-persistent -y

Saat instalasi iptables-persistent, pilih YES untuk menyimpan rule ipv4 & ipv6.

Masukkan Rule NAT:

# NAT keluar melalui ens33 (interface internet)
root@fw:~# iptables -t nat -A POSTROUTING -o ens33 -j MASQUERADE
# Simpan agar permanen
root@fw:~# netfilter-persistent save
4

DHCP Server (Untuk Client)

Sesuai soal, interface Client harus DHCP.

1. Install Paket

root@fw:~# apt install isc-dhcp-server -y

Wajar jika muncul pesan "Failed" saat install. Lanjut saja.

2. Tentukan Interface

root@fw:~# nano /etc/default/isc-dhcp-server
INTERFACESv4="ens39" # Sesuaikan dengan interface Client Anda

3. Konfigurasi Subnet

root@fw:~# nano /etc/dhcp/dhcpd.conf

Scroll ke paling bawah, tambahkan:

subnet 10.10.10.0 netmask 255.255.255.0 { range 10.10.10.10 10.10.10.100; option routers 10.10.10.1; option domain-name-servers 8.8.8.8; }

4. Restart DHCP

root@fw:~# systemctl restart isc-dhcp-server
root@fw:~# systemctl status isc-dhcp-server

Troubleshooting (Jika Error)

Masalah 1: "No DHCPOFFERS received" pada ens33

Penyebab: VM tidak tersambung ke WiFi fisik laptop Anda.
Solusi: Buka Virtual Network Editor > Klik VMnet0 > Ganti "Automatic" menjadi nama WiFi Adapter Laptop Anda > Restart Networking di Debian.

Masalah 2: "Job for isc-dhcp-server failed"

Penyebab: Salah ketik di file dhcpd.conf (kurang titik koma ;) atau salah nama interface di /etc/default/isc-dhcp-server.
Solusi: Cek nama interface dengan ip link show. Pastikan namanya sama persis (misal: ens39) dengan yang Anda tulis di config. Cek juga tanda kurung kurawal { } di file config.

Masalah 3: Ping Google Gagal (Temporary Failure in Name Resolution)

Penyebab: DNS belum tersetting atau koneksi internet putus.
Solusi: Cek file nano /etc/resolv.conf. Pastikan ada baris nameserver 8.8.8.8. Jika tidak ada, tambahkan.
