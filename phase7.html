High Availability

High Availability Proxy

Menggantikan Nginx Load Balancer dengan HAProxy (sesuai standar industri dan soal LKS) untuk membagi beban trafik ke Web01 dan Web02 secara adil.

Target Configuration

Peran Router (fw):

  • IP Public: 172.16.10.1 (DMZ GW)
  • Backend 1: 172.16.10.10 (Web01)
  • Backend 2: 172.16.10.20 (Web02)
  • Algorithm: Round Robin (Gantian)

Persiapan Wajib:

PERINGATAN: Di Fase 3 kita menginstall Nginx di Router.
Sekarang kita harus MEMATIKAN NGINX tersebut agar Port 80 bisa dipakai oleh HAProxy.
1

Matikan Nginx Lama

Langkah Kunci: Mencegah bentrok Port 80.

Lakukan ini di Router (fw) sebelum install HAProxy:

root@fw:~# systemctl stop nginx
root@fw:~# systemctl disable nginx
# Pastikan port 80 kosong:
root@fw:~# netstat -tulpn | grep :80

*Jika perintah netstat tidak muncul apa-apa, berarti port 80 aman (kosong).

2

Install HAProxy

Paket Load Balancer yang ringan dan stabil.

root@fw:~# apt install haproxy -y
3

Konfigurasi Load Balancing

Otak dari sistem HAProxy.

Edit File Config

root@fw:~# nano /etc/haproxy/haproxy.cfg

Tambahkan baris berikut di PALING BAWAH file:

# 1. MONITORING (Bonus Nilai Plus) listen stats bind :8080 stats enable stats uri /monitor stats auth admin:123 # 2. FRONTEND (Pintu Masuk) frontend http_front bind *:80 default_backend http_back # 3. BACKEND (Tujuan Server) backend http_back balance roundrobin server web01 172.16.10.10:80 check server web02 172.16.10.20:80 check

Restart Service

root@fw:~# systemctl restart haproxy
root@fw:~# systemctl status haproxy

Pengujian (Laptop Client)

1. Tes Load Balancing (Browser)

  • Buka Browser, akses: http://www.lks2026.id
  • Tekan Refresh (F5) berkali-kali.
  • Target: Tulisan harus berubah antara "Ini Server WEB 01" dan "Ini Server WEB 02".

2. Tes Failover (Simulasi Server Mati)

  • Matikan VM web01 (Suspend/Poweroff).
  • Refresh browser lagi.
  • Target: Website tetap jalan, tapi HANYA menampilkan "WEB 02". (HAProxy otomatis membuang web01 dari daftar).

3. Cek Monitoring (Nilai Plus)

Akses: http://172.16.10.1:8080/monitor

Login: admin | Password: 123. Anda akan melihat tabel status server yang hijau/merah.

Troubleshooting

Error: "Job for haproxy.service failed"
Penyebab 1: Salah ketik di haproxy.cfg. Cek spasi/tab.
Penyebab 2: Port 80 masih dipakai Nginx. Pastikan sudah systemctl stop nginx.
Website tidak berubah (Selalu Web 01)
Coba pakai Mode Incognito (Ctrl+Shift+N) karena browser sering menyimpan cache. Atau gunakan curl 172.16.10.1 di CMD untuk hasil yang lebih akurat.
