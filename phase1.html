Membangun Fondasi Lab

Fase ini menentukan keberhasilan fase berikutnya. Kita akan membangun "kabel virtual" sesuai topologi soal agar server-server nanti bisa saling terhubung di jalur yang benar.

Konteks Soal (Mengapa kita melakukan ini?)

Kutipan Skenario:

"...Anda diminta membangun infrastruktur jaringan yang aman dengan memisahkan layanan publik (DMZ), layanan internal (INT), dan Client..."

Logika IP Address:

  • Soal minta Web Server IP: 172.16.10.10 -> Maka kita buat Network 172.16.10.0 (DMZ).
  • Soal minta Int Server IP: 192.168.10.10 -> Maka kita buat Network 192.168.10.0 (INT).
  • Soal minta Client IP: 10.10.10.x -> Maka kita buat Network 10.10.10.0 (Client).
1

Setting Virtual Network Editor

Langkah ini wajib dilakukan sebelum membuat VM apapun.

1. Buka Aplikasi dengan Akses Admin

Cari "Virtual Network Editor" di menu Start Windows, klik kanan -> Run as Administrator.

2. Aktifkan Mode Edit

Jika tombol-tombol terlihat abu-abu (mati), klik tombol Change Settings di pojok kanan bawah jendela.

3. Tambahkan Network Baru (Ulangi 3x)

Klik tombol Add Network..., pilih VMnet yang sesuai, lalu atur detailnya:

VMnet Mode Subnet IP DHCP Fungsi
VMnet0 Bridged - - Internet
VMnet10 Host-only 172.16.10.0 UNCHECK DMZ
VMnet11 Host-only 192.168.10.0 UNCHECK INT
VMnet12 Host-only 10.10.10.0 UNCHECK Client

4. Simpan

Klik Apply dan tunggu proses restart network driver selesai, lalu klik OK.

2

Instalasi Master VM (Detail)

Panduan langkah-demi-langkah instalasi OS Debian 12.

2.1 Setup Awal VM

Buat VM Baru di VMware dengan spesifikasi:

  • ISO Image: `debian-12.x.x-amd64-DVD-1.iso`
  • VM Name: `Debian-Master`
  • Disk Size: `20 GB` (Store virtual disk as a single file)
  • Memory: `1024 MB` (1 GB)
  • Network Adapter: Pilih `NAT` (Agar bisa konek internet saat install).

2.2 Booting & Bahasa

  • Pilih menu Install (Jangan Graphical Install agar proses lebih cepat & ringan).
  • Language: English
  • Location: Other -> Asia -> Indonesia.
  • Keyboard: American English.

2.3 Identitas & Akun

Hostname master
Domain Name lks.kabupaten
  • Root Password: 123 (Konfirmasi: 123)
  • Full Name/User: siswa
  • User Password: 123 (Konfirmasi: 123)

2.4 Partisi Disk

Pilih Guided - use entire disk -> Pilih Harddisk -> All files in one partition -> Finish partitioning and write changes to disk -> Pilih Yes.

2.5 Package Manager (Repo)

  • Scan extra installation media? -> No
  • Use a network mirror? -> Yes
  • Mirror country: Indonesia -> Pilih `kambing.ui.ac.id` atau `kartolo` (Pilih yang stabil).
  • HTTP Proxy: Kosongkan -> Continue.

2.6 SOFTWARE SELECTION (SANGAT PENTING!)

Salah di sini akan membuat VM berat. Saat muncul menu "Software selection", atur centang persis seperti ini:

[ ] Debian desktop environment
[ ] ... GNOME
[ ] ... Xfce
[ ] Web server
[*] SSH server   <-- WAJIB DICENTANG
[*] Standard system utilities   <-- WAJIB DICENTANG

2.7 Bootloader & Finish

Install GRUB boot loader? -> Yes -> Pilih device /dev/sda (bukan manual).
Tunggu selesai, lalu Reboot.

3

Konfigurasi Pasca-Install

Menyiapkan "Senjata Perang" di dalam VM Master.

3.1 Login Pertama & Akses Root

Login menggunakan user siswa, lalu masuk ke mode superuser.

master login: siswa
Password: 123
siswa@master:~$ su -
Password: 123
root@master:~# # Prompt berubah jadi pagar (#) artinya Root

3.2 Cek Koneksi & Repository

Pastikan VM bisa internet (karena adaptor NAT) dan update daftar paket.

root@master:~# ping google.com -c 3
root@master:~# nano /etc/apt/sources.list
# Pastikan baris 'deb cdrom' diberi tanda pagar (#) didepannya agar tidak minta CD.
# Simpan: Ctrl+O, Enter. Keluar: Ctrl+X
root@master:~# apt update

3.3 Install Paket Wajib (Tools)

Install semua alat yang dibutuhkan untuk LKS agar tidak perlu download lagi nanti.

root@master:~# apt install nano vim net-tools dnsutils iputils-ping curl wget zip unzip git -y

3.4 Aktifkan SSH Root Login (Efisiensi Waktu)

Secara default, Debian 12 memblokir login root via SSH. Kita buka agar config lebih cepat.

root@master:~# nano /etc/ssh/sshd_config
Cari baris:
#PermitRootLogin prohibit-password

Ubah menjadi (Hapus pagar & ganti text):
PermitRootLogin yes
root@master:~# systemctl restart ssh

3.5 Finalisasi

Matikan VM. Master siap dikloning.

root@master:~# apt clean
root@master:~# poweroff

Checklist Selesai Fase 1

Virtual Network Editor memiliki VMnet10, 11, 12 (DHCP Off).
VM Master Debian terinstall (CLI Only/Tanpa GUI).
Perintah `ifconfig` dan `dig` bisa dijalankan.
VM Master dalam kondisi mati (Poweroff).
