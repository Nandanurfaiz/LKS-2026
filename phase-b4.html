OSPF Routing Protocol

Menghubungkan Router Pusat (R1) dan Router Cabang (R2) secara otomatis. Router akan saling "bertukar peta" untuk mencari jalur tercepat.

Kenapa OSPF?

Static Routing itu capek. Jika kabel putus, admin harus setting ulang manual. OSPF (Dynamic) mendeteksi jalur mati dan mencari jalan alternatif secara otomatis dalam hitungan detik.

Area 0 = Backbone (Area Wajib)

Rumus Wildcard Mask

OSPF tidak pakai Subnet Mask, tapi Wildcard Mask.

Total: 255.255.255.255
Mask : 255.255.255.0 (-)
----------------------
Wild : 0.0.0.255

Contoh /30 (255.255.255.252) -> Wildcard: 0.0.0.3

Topologi Routing

Router Interface Network (IP) Wildcard Mask
R1 (Pusat) LAN (VLANs) 192.168.10.0 & 20.0 0.0.0.255
Serial0/0/0 (WAN) 10.10.10.0/30 0.0.0.3
Loopback 0 1.1.1.1 (Router ID) -
R2 (Cabang) Gigabit0/0 (LAN) 192.168.30.0/24 0.0.0.255
Serial0/0/0 (WAN) 10.10.10.0/30 0.0.0.3

Skenario: R1 dan R2 terhubung lewat kabel Serial (WAN Link).

1

Konfigurasi R1 (Head Office)

Mendaftarkan semua network milik R1 ke OSPF.

R1(config)# router ospf 1 ! 1. Tentukan Identitas (Router-ID) R1(config-router)# router-id 1.1.1.1 ! 2. Umumkan Network LAN (VLAN 10 & 20) R1(config-router)# network 192.168.10.0 0.0.0.255 area 0 R1(config-router)# network 192.168.20.0 0.0.0.255 area 0 ! 3. Umumkan Network WAN (Link ke R2) R1(config-router)# network 10.10.10.0 0.0.0.3 area 0 ! 4. Security: Passive Interface R1(config-router)# passive-interface g0/0.10 R1(config-router)# passive-interface g0/0.20
2

Konfigurasi R2 (Branch Office)

Mendaftarkan network cabang agar bisa diakses pusat.

R2(config)# router ospf 1 R2(config-router)# router-id 2.2.2.2 ! 1. Umumkan LAN Cabang R2(config-router)# network 192.168.30.0 0.0.0.255 area 0 ! 2. Umumkan Network WAN (Link ke R1) R2(config-router)# network 10.10.10.0 0.0.0.3 area 0 ! 3. Security R2(config-router)# passive-interface g0/0
3

Internet & Default Route (Advanced)

Menyebarkan jalur internet dari R1 ke seluruh jaringan OSPF.

Asumsi: R1 terhubung ke ISP lewat IP 202.10.10.1. Kita buat R2 bisa internetan lewat R1.

! Di Router 1 (Pusat) ! 1. Buat rute statis ke ISP ("Pintu Keluar") R1(config)# ip route 0.0.0.0 0.0.0.0 202.10.10.1 ! 2. Masukkan ke OSPF (Sebarkan Info) R1(config)# router ospf 1 R1(config-router)# default-information originate

Verifikasi OSPF

1. Cek Tetangga (Neighbor)

R1# show ip ospf neighbor

Pastikan statusnya FULL/DR atau FULL/BDR atau FULL/-.
Jika statusnya Init atau 2WAY terus menerus, cek kabel atau MTU.

2. Cek Tabel Routing (Routing Table)

R2# show ip route

Di R2, cari kode O (OSPF).
R2 harusnya punya rute ke 192.168.10.0 (VLAN 10 R1) dan O*E2 0.0.0.0/0 (Jalur Internet).

3. Ping Lintas Cabang

PC Cabang (192.168.30.x) ➔ Ping ➔ PC Siswa (192.168.10.x).
Hasil: Reply.
