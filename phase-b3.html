Fase B3 High Availability
| EtherChannel & STP
Cisco IOS

Redundansi Jaringan

Menggabungkan kabel fisik untuk kecepatan ganda dan mengatur lalu lintas agar tidak terjadi "kemacetan abadi" (Looping).

Pahami Konsepnya

EtherChannel (LACP)

Mengikat 2 kabel fisik menjadi 1 kabel logis.
Manfaat: Speed naik 2x lipat (200 Mbps) & jika 1 kabel putus, koneksi tetap jalan.

Spanning Tree (STP)

Polisi lalu lintas anti-looping. Tanpa STP, data akan berputar selamanya di antara kabel redundant sampai switch hang. STP memblokir salah satu jalur sebagai cadangan.

Persiapan Fisik (Wajib)

Sebelum konfigurasi, tambahkan 1 kabel Cross-Over lagi antara switch.
Hubungkan: SW-1 Port Fa0/3 <--> SW-2 Port Fa0/3.
(Sekarang ada 2 kabel menghubungkan switch: Fa0/2 dan Fa0/3).

1

Konfigurasi EtherChannel (LACP)

Lakukan langkah ini di KEDUA Switch (SW-1 dan SW-2).

! Pilih kedua port penghubung SW-1(config)# interface range fa0/2-3 ! 1. Bungkus menjadi Channel Group 1 SW-1(config-if-range)# channel-group 1 mode active ! 2. Nyalakan port (Just in case) SW-1(config-if-range)# no shutdown SW-1(config-if-range)# exit ! 3. Konfigurasi Trunk pada Interface LOGICAL (Port-Channel) SW-1(config)# interface port-channel 1 SW-1(config-if)# switchport mode trunk SW-1(config-if)# switchport trunk native vlan 99
Tips: Jangan lupa lakukan hal yang sama persis di SW-2. Konfigurasi Trunk sekarang pindah ke interface port-channel 1, bukan lagi di fisik Fa0/2.
2

Optimasi Spanning Tree (Rapid-PVST)

Mempercepat waktu pemulihan jika kabel putus (dari 50 detik jadi < 2 detik).

A. Di SW-1 (Switch Utama/Root)

SW-1(config)# spanning-tree mode rapid-pvst SW-1(config)# spanning-tree vlan 10,20,99 root primary

B. Di SW-2 (Switch Cadangan)

SW-2(config)# spanning-tree mode rapid-pvst SW-2(config)# spanning-tree vlan 10,20,99 root secondary
3

Port Security (Bonus Point)

Mengunci port agar hanya bisa dipakai PC yang terdaftar.

Terapkan pada port Access (PC Siswa & Guru) di kedua switch:

SW-1(config)# interface range fa0/5, fa0/10 SW-1(config-if-range)# switchport mode access SW-1(config-if-range)# switchport port-security SW-1(config-if-range)# switchport port-security maximum 1 SW-1(config-if-range)# switchport port-security mac-address sticky SW-1(config-if-range)# switchport port-security violation restrict

Verifikasi Hasil

1. Cek EtherChannel
SW-1# show etherchannel summary

Cari kode (SU) di samping Po1.
S = Layer 2, U = In Use (Aktif).
Protocol harus LACP.

2. Cek Spanning Tree
SW-1# show spanning-tree

Di SW-1 harus ada tulisan: "This bridge is the root".
Interface yang muncul adalah Po1 (bukan Fa0/2 atau Fa0/3 lagi).
