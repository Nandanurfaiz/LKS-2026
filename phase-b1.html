Modul B

VLAN Management

Kita akan menggunakan protokol VTP (VLAN Trunking Protocol) agar tidak perlu membuat VLAN di setiap switch secara manual. Cukup buat di Server, otomatis menyebar ke Client.

1

Konfigurasi VTP Server

Pusat database VLAN ada di sini.

Klik Switch 1 di Packet Tracer, masuk ke tab CLI.

Switch> enable
Switch# configure terminal
Switch(config)# hostname SW1
! Setting VTP Domain & Mode
SW1(config)# vtp domain lks2026
SW1(config)# vtp mode server
SW1(config)# vtp password cisco
SW1(config)# exit
2

Membuat VLAN

Hanya dilakukan di VTP Server.

Masih di Switch 1. Buat dua VLAN sesuai soal.

SW1(config)# vlan 10
SW1(config-vlan)# name SISWA
SW1(config-vlan)# exit

SW1(config)# vlan 20
SW1(config-vlan)# name GURU
SW1(config-vlan)# exit
3

Konfigurasi Trunk

Jalan tol agar VLAN bisa menyeberang antar switch.

Di Switch 1

! Link ke Router & SW2
SW1(config)# int range f0/1-2, g0/1
SW1(config-if-range)# switchport mode trunk
SW1(config-if-range)# exit

Di Switch 2

Switch> en
Switch# conf t
Switch(config)# hostname SW2
SW2(config)# int range f0/1-2
SW2(config-if-range)# switchport mode trunk
Penting: Setelah trunk aktif, VTP Advertisement akan terkirim. VLAN di SW1 akan otomatis muncul di SW2.
4

Konfigurasi VTP Client

Menerima database VLAN dari Server.

Buka Switch 2. Setting menjadi mode Client.

SW2(config)# vtp domain lks2026
SW2(config)# vtp mode client
SW2(config)# vtp password cisco
SW2(config)# do show vlan brief

*Cek hasil show vlan brief. Jika sukses, VLAN 10 & 20 sudah ada disitu tanpa kita ketik manual.

5

Assign Port ke VLAN

Menghubungkan PC ke VLAN yang sesuai.

Lakukan ini di KEDUA SWITCH (SW1 & SW2) agar PC bisa connect.

! Port F0/10 untuk Siswa (VLAN 10)
SW(config)# int f0/10
SW(config-if)# switchport mode access
SW(config-if)# switchport access vlan 10

! Port F0/20 untuk Guru (VLAN 20)
SW(config)# int f0/20
SW(config-if)# switchport mode access
SW(config-if)# switchport access vlan 20
