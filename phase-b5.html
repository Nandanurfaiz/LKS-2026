Keamanan & Translasi Jaringan

Tahap akhir: Menyembunyikan IP Lokal agar bisa akses internet (NAT) dan membatasi akses liar menggunakan Access Control List (ACL).

Pahami Konsepnya

NAT Overload (PAT)

Seperti resepsionis kantor. Ratusan karyawan (IP Lokal) bisa kirim surat keluar hanya menggunakan 1 alamat kantor pusat (IP Publik/WAN). Hemat IP Publik.

ACL (Access Control List)

Daftar tamu. Satpam (Router) akan mengecek setiap paket. Jika nama ada di daftar "Deny", maka paket dibuang. Jika "Permit", paket boleh lewat.

1

Konfigurasi NAT Overload

Mengizinkan VLAN 10 dan 20 akses ke Internet (Loopback 0).

A. Tentukan Pintu Masuk & Keluar

R1(config)# interface range fa0/0.10, fa0/0.20 R1(config-if-range)# ip nat inside R1(config-if-range)# exit R1(config)# interface loopback 0 R1(config-if)# ip nat outside R1(config-if)# exit

B. Buat Aturan NAT (Siapa yang boleh internetan?)

! 1. Buat Access List untuk mencatat IP Lokal R1(config)# access-list 1 permit 192.168.10.0 0.0.0.255 R1(config)# access-list 1 permit 192.168.20.0 0.0.0.255 ! 2. Aktifkan NAT Overload R1(config)# ip nat inside source list 1 interface loopback 0 overload
2

Amankan Router (SSH)

Agar admin tidak perlu colok kabel console (remote setting), tapi tetap terenkripsi.

! 1. Syarat SSH: Harus ada domain name R1(config)# ip domain-name lks2026.id ! 2. Buat Kunci Enkripsi (Pilih 1024 bits) R1(config)# crypto key generate rsa ! 3. Buat User Admin R1(config)# username admin privilege 15 secret lks123 ! 4. Terapkan pada Jalur Remote (VTY) R1(config)# line vty 0 4 R1(config-line)# login local R1(config-line)# transport input ssh R1(config-line)# exit R1(config)# ip ssh version 2
3

ACL Restriction (Pembatasan)

Skenario: PC Siswa (VLAN 10) DILARANG ping ke Router (192.168.10.1), tapi boleh akses lain.

! Buat Access List Extended (Nomor 100-199) R1(config)# access-list 100 deny icmp 192.168.10.0 0.0.0.255 host 192.168.10.1 R1(config)# access-list 100 permit ip any any ! Pasang Satpam (ACL) di Pintu Masuk VLAN 10 R1(config)# interface fa0/0.10 R1(config-subif)# ip access-group 100 in

Uji Coba Terakhir

1. Tes NAT (Ping Internet)

Dari PC Siswa, ping ke 8.8.8.8. Harus Reply.

R1# show ip nat translations

Harus muncul daftar translasi (Inside Global vs Inside Local).

2. Tes ACL (Ping Gateway)

Dari PC Siswa, ping ke 192.168.10.1.

Hasil harus: Destination host unreachable / Request Timed Out.

3. Tes SSH

Dari PC Guru, buka "Telnet/SSH Client". Pilih SSH. IP: 192.168.20.1. User: admin.
