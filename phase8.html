Modul Security

Advanced Routing & Security

Menambahkan **Virtual IP** (untuk persiapan High Availability) dan mengamankan Router dengan **IPTables**.

Skenario Keamanan

Target Konfigurasi:

  • Virtual IP: 172.16.10.254 (Secondary IP).
  • Ping: Router MENOLAK ping dari Client.
  • SSH: Hanya ADMIN (Lokal) yang boleh remote.

Mengapa Virtual IP?

Di kisi-kisi diminta "Virtual IP". Ini teknik membuat 1 kabel LAN memiliki 2 alamat IP. Berguna jika IP utama mati, IP virtual bisa pindah ke router cadangan (Failover).

1

Virtual IP (IP Alias)

Materi Baru (Wajib!)

Kita akan menambahkan IP 172.16.10.254 pada interface DMZ (misal: ens34 atau ens37) tanpa menghilangkan IP lama.

1. Cek Nama Interface

root@fw:~# ip a

2. Tambahkan Config Alias

root@fw:~# nano /etc/network/interfaces

Tambahkan baris ini di paling bawah (Sesuaikan ens34 dengan nama interface DMZ Anda!):

auto ens34:0 iface ens34:0 inet static address 172.16.10.254/24

3. Aktifkan & Tes

root@fw:~# systemctl restart networking
root@fw:~# ping 172.16.10.254

Jika Reply, berarti Router Anda sekarang punya 2 identitas!

2

Cek Status Firewall

Persiapan IPTables.

root@fw:~# apt install iptables-persistent -y
root@fw:~# iptables -L -n -v
3

Blokir Ping (ICMP)

Router tidak boleh menjawab Ping dari Client.

Penerapan Rule

# Ganti 'ens33' dengan interface arah Client/Internet jika perlu
root@fw:~# iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

Validasi

Dari Laptop Client: ping 172.16.10.1 -> Harus Request Timed Out.

4

Amankan SSH (Port 22)

Whitelist IP Admin.

# 1. Izinkan Network Lokal (Admin)
root@fw:~# iptables -A INPUT -p tcp --dport 22 -s 172.16.10.0/24 -j ACCEPT
root@fw:~# iptables -A INPUT -p tcp --dport 22 -s 192.168.10.0/24 -j ACCEPT

# 2. Blokir Sisanya
root@fw:~# iptables -A INPUT -p tcp --dport 22 -j DROP
5

Simpan Permanen

Wajib agar tidak hilang saat reboot.

root@fw:~# netfilter-persistent save
