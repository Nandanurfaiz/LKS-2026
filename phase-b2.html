Fase B2 Layer 3 Routing
| Router on a Stick & DHCP
Cisco IOS

Menghubungkan VLAN (Layer 3)

Membuat "Jembatan Virtual" di dalam Router agar VLAN Siswa dan Guru bisa saling komunikasi, serta membagikan IP Address secara otomatis.

Pahami Konsepnya

Router on a Stick (RoaS)

Alih-alih menarik 3 kabel fisik (boros), kita menggunakan 1 kabel Trunk fisik yang dipecah menjadi banyak Sub-interface virtual (Fa0/0.10, Fa0/0.20).

DHCP Server

Layanan pemberi IP otomatis. Router akan menjadi "Bos" yang meminjamkan IP ke PC Siswa dan Guru agar kita tidak perlu setting manual satu per satu.

Tabel Alokasi IP (Gateway):

VLANGateway IPNetwork

1

Aktifkan Interface Fisik

Menyalakan port utama Router yang terhubung ke Switch.

Masuk ke Router R1:

Router> enable Router# configure terminal Router(config)# hostname R1 ! Masuk ke port fisik (sesuai cabling Fa0/0) R1(config)# interface fa0/0 R1(config-if)# no shutdown R1(config-if)# exit
PENTING: Interface fisik (Fa0/0) harus polosan (no ip address). IP address akan dipasang di anak-anaknya (sub-interface).
2

Konfigurasi Sub-Interface (RoaS)

Memecah port fisik menjadi gateway virtual untuk tiap VLAN.

! 1. Gateway VLAN 10 (Siswa) R1(config)# interface fa0/0.10 R1(config-subif)# encapsulation dot1q 10 R1(config-subif)# ip address 192.168.10.1 255.255.255.0 R1(config-subif)# exit ! 2. Gateway VLAN 20 (Guru) R1(config)# interface fa0/0.20 R1(config-subif)# encapsulation dot1q 20 R1(config-subif)# ip address 192.168.20.1 255.255.255.0 R1(config-subif)# exit ! 3. Gateway Native VLAN 99 (Management) R1(config)# interface fa0/0.99 R1(config-subif)# encapsulation dot1q 99 native R1(config-subif)# ip address 192.168.99.1 255.255.255.0 R1(config-subif)# exit
3

Setup DHCP Server

Membagikan IP otomatis ke PC Client.

! Amankan IP Gateway agar tidak diberikan ke Client R1(config)# ip dhcp excluded-address 192.168.10.1 R1(config)# ip dhcp excluded-address 192.168.20.1 ! Buat Pool untuk SISWA R1(config)# ip dhcp pool POOL_SISWA R1(dhcp-config)# network 192.168.10.0 255.255.255.0 R1(dhcp-config)# default-router 192.168.10.1 R1(dhcp-config)# dns-server 8.8.8.8 R1(dhcp-config)# exit ! Buat Pool untuk GURU R1(config)# ip dhcp pool POOL_GURU R1(dhcp-config)# network 192.168.20.0 255.255.255.0 R1(dhcp-config)# default-router 192.168.20.1 R1(dhcp-config)# dns-server 8.8.8.8 R1(dhcp-config)# exit

Verifikasi Hasil

1. Request DHCP di PC

Klik PC Client > Desktop > IP Configuration > Pilih DHCP.
Pastikan mendapat IP 192.168.xx.xx dan Gateway yang benar.

2. Ping Lintas VLAN

Dari PC Siswa (VLAN 10), ping ke PC Guru (VLAN 20).
Status awal mungkin Request Timed Out (ARP), coba lagi sampai Reply.
