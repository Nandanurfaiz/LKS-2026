Modul B

Routing & Redundancy

Menghubungkan VLAN agar bisa saling komunikasi (Inter-VLAN), membagikan IP otomatis (DHCP), dan menyiapkan Gateway Cadangan (HSRP) agar jaringan tidak mati total saat router utama rusak.

1

Inter-VLAN Routing

Metode "Router on a Stick" pada HQ.

Buka Router HQ. Kita akan memecah port G0/1 menjadi beberapa sub-interface.

HQ> en
HQ# conf t
! Aktifkan interface fisik dulu
HQ(config)# int g0/1
HQ(config-if)# no shut
HQ(config-if)# exit

! Sub-interface VLAN 10
HQ(config)# int g0/1.10
HQ(config-subif)# encapsulation dot1q 10
HQ(config-subif)# ip add 192.168.10.1 255.255.255.0
HQ(config-subif)# exit

! Sub-interface VLAN 20
HQ(config)# int g0/1.20
HQ(config-subif)# encapsulation dot1q 20
HQ(config-subif)# ip add 192.168.20.1 255.255.255.0
HQ(config-subif)# exit
2

DHCP Server

Agar PC Client dapat IP otomatis.

Masih di Router HQ. Kita buat dua kolam IP (Pool).

! Exclude IP Gateway & Virtual Gateway (HSRP)
HQ(config)# ip dhcp excluded-address 192.168.10.1 192.168.10.10
HQ(config)# ip dhcp excluded-address 192.168.10.254

! Pool Siswa
HQ(config)# ip dhcp pool SISWA
dhcp-config# network 192.168.10.0 255.255.255.0
dhcp-config# default-router 192.168.10.254 ! Arahkan ke IP Virtual (HSRP)
dhcp-config# dns-server 8.8.8.8
dhcp-config# exit
Tips: Default Router kita set ke .254 (IP Virtual HSRP) bukan .1. Ini kuncinya agar jika Router HQ mati, PC tetap bisa internetan lewat Router Backup!

*Ulangi langkah di atas untuk VLAN 20 (GURU).

3

Konfigurasi HSRP

First Hop Redundancy Protocol.

Siapkan satu router lagi, beri nama Router Backup. Hubungkan ke Switch 2.

A. Di Router HQ (Primary)

HQ(config)# int g0/1.10
HQ(config-subif)# standby 10 ip 192.168.10.254 ! IP Hantu
HQ(config-subif)# standby 10 priority 150 ! Priority tinggi = Jadi Bos (Active)
HQ(config-subif)# standby 10 preempt ! Rebut kembali jika hidup lagi

B. Di Router Backup (Secondary)

Backup(config)# int g0/1.10
Backup(config-subif)# ip add 192.168.10.2 255.255.255.0
Backup(config-subif)# standby 10 ip 192.168.10.254
Backup(config-subif)# standby 10 priority 100 ! Priority rendah = Jadi Cadangan
Backup(config-subif)# standby 10 preempt
4

Verifikasi Failover

Uji coba matikan router utama.

1. Cek Status HSRP

HQ# show standby brief
State: Active

Backup# show standby brief
State: Standby

2. Simulasi Kabel Putus

  • Buka PC Client, lakukan ping 192.168.10.254 -t (ping terus menerus).
  • Matikan Router HQ (atau cabut kabel G0/1).
  • Ping mungkin RTO 1-2 kali, lalu Reply kembali.
  • Itu artinya Router Backup sukses mengambil alih!
