Router on a Stick

Menghubungkan VLAN yang terisolasi menggunakan satu kabel fisik ke Router (Sub-interface) dan membagikan IP Address secara otomatis.

Masalah: Isolasi VLAN

Di Fase B1, PC Siswa (VLAN 10) tidak bisa Ping ke PC Guru (VLAN 20). Kenapa? Karena Switch adalah perangkat Layer 2 yang bekerja berdasarkan MAC Address. Ia tidak bisa memindahkan paket antar Network (IP) yang berbeda.

Solusi: Kita butuh "Polisi Lalu Lintas" (Router) di Layer 3.

Konsep: Sub-Interfaces

Alih-alih menarik 3 kabel dari Switch ke Router (boros port), kita cukup pakai 1 kabel Trunk. Di dalam Router, kabel fisik ini kita "belah" secara virtual menjadi anak-anak interface (g0/0.10, g0/0.20).

Inilah yang disebut "Router on a Stick".

Rencana IP Addressing

VLAN Network ID Gateway (Router IP) Range DHCP
10 (SISWA) 192.168.10.0/24 192.168.10.1 .10 - .254
20 (GURU) 192.168.20.0/24 192.168.20.1 .10 - .254
99 (NATIVE) 192.168.99.0/24 192.168.99.1 Management Only
1

Persiapan Switch (Uplink)

Menyiapkan port yang mengarah ke Router sebagai Trunk.

Asumsi: Router R1 terhubung ke Port Gigabit0/2 di SW-1.

SW-1(config)# interface gi0/2 SW-1(config-if)# switchport mode trunk SW-1(config-if)# switchport trunk native vlan 99 SW-1(config-if)# switchport trunk allowed vlan 10,20,99 SW-1(config-if)# no shutdown
2

Konfigurasi Router (Sub-Interfaces)

Membuat gateway virtual untuk masing-masing VLAN.

Router> enable Router# configure terminal Router(config)# hostname R1 ! 1. Nyalakan Interface Fisik dulu (PENTING) R1(config)# interface g0/0 R1(config-if)# no shutdown R1(config-if)# exit ! 2. Gateway VLAN 10 R1(config)# interface g0/0.10 R1(config-subif)# encapsulation dot1q 10 R1(config-subif)# ip address 192.168.10.1 255.255.255.0 R1(config-subif)# exit ! 3. Gateway VLAN 20 R1(config)# interface g0/0.20 R1(config-subif)# encapsulation dot1q 20 R1(config-subif)# ip address 192.168.20.1 255.255.255.0 R1(config-subif)# exit ! 4. Gateway Native VLAN 99 (Trap LKS) R1(config)# interface g0/0.99 R1(config-subif)# encapsulation dot1q 99 native R1(config-subif)# ip address 192.168.99.1 255.255.255.0
Tips LKS: Jangan lupa no shutdown di interface fisik induk (g0/0). Jika induk mati, semua sub-interface juga mati.
3

Konfigurasi DHCP Server

Menjadikan Router sebagai pemberi IP otomatis ke Client.

! Exclude IP Gateway (Agar tidak bentrok) R1(config)# ip dhcp excluded-address 192.168.10.1 R1(config)# ip dhcp excluded-address 192.168.20.1 ! Pool untuk SISWA R1(config)# ip dhcp pool POOL_SISWA R1(dhcp-config)# network 192.168.10.0 255.255.255.0 R1(dhcp-config)# default-router 192.168.10.1 R1(dhcp-config)# dns-server 8.8.8.8 R1(dhcp-config)# exit ! Pool untuk GURU R1(config)# ip dhcp pool POOL_GURU R1(dhcp-config)# network 192.168.20.0 255.255.255.0 R1(dhcp-config)# default-router 192.168.20.1 R1(dhcp-config)# dns-server 8.8.8.8 R1(dhcp-config)# exit

Verifikasi Akhir

1. Request DHCP di PC

Di PC Desktop Packet Tracer > IP Configuration > Pilih DHCP.
Pastikan mendapat IP 192.168.10.x (Siswa) atau 192.168.20.x (Guru).

2. Ping Lintas VLAN

Coba ping dari PC Siswa ke PC Guru.
Status awal mungkin Request Timed Out (ARP Process), coba lagi kedua kali harus Reply.

Kembali ke B1 Lanjut ke Fase B3 (Redundancy)
