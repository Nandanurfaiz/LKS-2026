Menghubungkan VLAN (Layer 3)

Fokus konfigurasi pada Router R1. Kita akan membuat "Jembatan Virtual" agar VLAN Siswa dan Guru bisa saling komunikasi, serta membagikan IP Address secara otomatis.

Pahami Konsepnya

Router on a Stick (RoaS)

Alih-alih menarik 3 kabel fisik (boros), kita menggunakan 1 kabel fisik di R1 yang dipecah menjadi banyak Sub-interface virtual (Fa0/0.10, Fa0/0.20) untuk melayani tiap VLAN.

DHCP Server

R1 akan menjadi "Bos" yang meminjamkan IP ke PC Client secara otomatis, jadi kita tidak perlu setting IP manual satu per satu.

Tabel Alokasi IP (Gateway R1):

VLANSub-InterfaceGateway IP
10 (Siswa)Fa0/0.10192.168.10.1
20 (Guru)Fa0/0.20192.168.20.1

1

Aktifkan Interface Fisik

Target Device: R1 (Router)

Masuk ke CLI Router. Pastikan kabel fisik Fa0/0 menyala.

Router> enable Router# configure terminal Router(config)# hostname R1 ! Masuk ke port fisik utama R1(config)# interface fa0/0 R1(config-if)# no shutdown R1(config-if)# exit
PERINGATAN: Jangan beri IP Address di interface fisik (Fa0/0) ini! Biarkan dia polosan. IP akan kita pasang di sub-interface langkah berikutnya.
2

Konfigurasi Sub-Interface (RoaS)

Target Device: R1 (Router)

! 1. Gateway VLAN 10 (Siswa) R1(config)# interface fa0/0.10 R1(config-subif)# encapsulation dot1q 10 R1(config-subif)# ip address 192.168.10.1 255.255.255.0 R1(config-subif)# exit ! 2. Gateway VLAN 20 (Guru) R1(config)# interface fa0/0.20 R1(config-subif)# encapsulation dot1q 20 R1(config-subif)# ip address 192.168.20.1 255.255.255.0 R1(config-subif)# exit ! 3. Gateway Native VLAN 99 (Management) R1(config)# interface fa0/0.99 R1(config-subif)# encapsulation dot1q 99 native R1(config-subif)# ip address 192.168.99.1 255.255.255.0 R1(config-subif)# exit
3

Setup DHCP Server

Target Device: R1 (Router)

Membuat router meminjamkan IP secara otomatis ke PC.

! Langkah A: Amankan IP Gateway (Supaya tidak dipinjamkan ke orang lain) R1(config)# ip dhcp excluded-address 192.168.10.1 R1(config)# ip dhcp excluded-address 192.168.20.1 ! Langkah B: Buat Kolam IP untuk SISWA R1(config)# ip dhcp pool POOL_SISWA R1(dhcp-config)# network 192.168.10.0 255.255.255.0 R1(dhcp-config)# default-router 192.168.10.1 R1(dhcp-config)# dns-server 8.8.8.8 R1(dhcp-config)# exit ! Langkah C: Buat Kolam IP untuk GURU R1(config)# ip dhcp pool POOL_GURU R1(dhcp-config)# network 192.168.20.0 255.255.255.0 R1(dhcp-config)# default-router 192.168.20.1 R1(dhcp-config)# dns-server 8.8.8.8 R1(dhcp-config)# exit

Verifikasi Hasil

1. Request DHCP di PC Client

Buka PC Client > Desktop > IP Configuration > Pilih DHCP.
Pastikan mendapat IP 192.168.xx.xx dan Gateway yang benar.

2. Ping Lintas VLAN

Dari PC Siswa (VLAN 10), ping ke PC Guru (VLAN 20).
Status awal mungkin Request Timed Out (karena proses ARP), coba lagi sampai Reply.
