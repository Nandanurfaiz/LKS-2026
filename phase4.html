Pembuktian Kebenaran

Sebelum masuk ke layanan tingkat lanjut (Mail & LDAP), kita wajib memastikan fondasi jaringan (Router, DNS, Web, Load Balancer) sudah 100% solid.

Misi Pengujian

Target Client:

  • VM Name: Client / Budi-Clt
  • OS: Windows 11 (Sesuai Kisi-kisi)
  • Network: VMnet12 (Wajib!)
  • IP Address: DHCP (Otomatis)

Checklist Sukses:

  • Mendapat IP 10.10.10.x.
  • Ping ke Gateway 10.10.10.1 (OK).
  • Buka Web www.lks2026.id (OK).
  • Refresh Web -> Konten berubah (LB OK).

Persiapan VM (Install Windows)

Langkah awal membuat mesin Client.

1. Buat VM Baru

Di VMware Workstation, klik File > New Virtual Machine.

  • Installer Disc Image: Pilih file ISO Windows 11 (atau Win 10/7 untuk latihan ringan).
  • Full Name: Siswa / Budi
  • Virtual Machine Name: Client-Windows
  • Disk Size: 60 GB (Store virtual disk as a single file).

2. Setting Network (SANGAT PENTING!)

Sebelum klik Finish, klik tombol Customize Hardware.

Pilih Network Adapter -> Ganti ke Custom: VMnet12.

*Jika salah network (misal NAT/Bridged), Client tidak akan dapat IP dari Router Debian kita.

3. Jalankan Instalasi

Klik Finish, lalu jalankan VM. Ikuti proses instalasi Windows sampai masuk ke Desktop.

1

Konfigurasi IP (DHCP)

Meminta IP otomatis dari Router.

1. Buka Network Connections

Di Windows Client, tekan Win + R, ketik ncpa.cpl, lalu Enter.

2. Setting IPv4

  • Klik Kanan Adapter Ethernet > Properties.
  • Double-klik Internet Protocol Version 4 (TCP/IPv4).
  • Pastikan pilih: Obtain an IP address automatically.
  • Pastikan pilih: Obtain DNS server address automatically.
  • Klik OK dan OK.
2

Tes Koneksi (Ping & IP)

Memastikan Router & DHCP bekerja.

1. Cek IP Address

Buka CMD di Windows, ketik:

C:\Users\Siswa> ipconfig
Target: IPv4 Address harus 10.10.10.x (antara 10-100).
Gateway harus 10.10.10.1.

2. Ping Gateway & Internet

C:\Users\Siswa> ping 10.10.10.1
C:\Users\Siswa> ping 8.8.8.8

*Ping 8.8.8.8 hanya akan Reply jika Laptop Asli Anda terhubung internet.

3

Tes DNS & Web (Final)

Momen penentuan keberhasilan.

1. Tes Domain (Nslookup)

Pastikan Client mengenal nama domain kita:

C:\Users\Siswa> nslookup www.lks2026.id
Server: UnKnown Address: 10.10.10.1 <-- DNS Server (Router) Name: www.lks2026.id Address: 172.16.10.1 <-- IP Load Balancer (Gateway DMZ)

2. Tes Web Browser & Load Balancing

Buka Chrome/Edge, akses http://www.lks2026.id.

Troubleshooting Akhir

Ping Reply, tapi Web tidak bisa dibuka
Solusi: Cek konfigurasi Nginx Load Balancer di Router. Pastikan `proxy_pass` mengarah ke IP `172.16.10.10` dan `172.16.10.20`. Cek juga apakah firewall (iptables) tidak memblokir port 80.
Hanya bisa akses IP, Domain gagal
Solusi: Masalah ada di DNS (Bind9). Cek file `db.lks`. Pastikan di Windows Client, DNS Server yang didapat adalah IP Router (10.10.10.1). Coba `ipconfig /flushdns` di CMD.
