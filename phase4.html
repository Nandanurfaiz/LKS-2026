Pembuktian (Laptop Fisik)

Anda menggunakan Strategi B. Kita akan menguji apakah Laptop Anda yang terhubung kabel LAN bisa mengakses Server Virtual yang ada di PC ini.

Misi Pengujian

Persiapan Hardware:

  • PC Server: Menjalankan VMware (Router fw).
  • Laptop Client: Windows Asli.
  • Koneksi: Kabel LAN (UTP) Direct/Switch.
  • Setting: Wi-Fi Laptop harus OFF.

Checklist Sukses:

  • Laptop dapat IP 10.10.10.x.
  • Ping ke Gateway 10.10.10.1 (OK).
  • Buka Web www.lks2026.id (OK).
1

Koneksi Fisik & Network

Persiapan di Laptop Client.

1. Matikan Wi-Fi

Agar tidak bentrok gateway, matikan Wi-Fi di Laptop Anda sekarang.

2. Hubungkan Kabel LAN

Colokkan kabel LAN dari Laptop ke Port LAN PC Server. Pastikan lampu indikator LAN menyala.

3. Setting IP (DHCP)

Di Laptop Client:

  • Tekan Win + R, ketik ncpa.cpl.
  • Klik Kanan Ethernet > Properties > IPv4.
  • Pilih Obtain an IP address automatically.
  • Klik OK.
2

Tes Koneksi (CMD Laptop)

Buka Command Prompt di Laptop Fisik.

1. Cek IP Address

Ketik di CMD Laptop:

C:\Users\LaptopSaya> ipconfig
Target: IPv4: 10.10.10.x | Gateway: 10.10.10.1

2. Ping Gateway

C:\Users\LaptopSaya> ping 10.10.10.1
3

Tes Layanan (Browser)

Pengujian Web & DNS dari Laptop.

1. Tes Domain (Nslookup)

Pastikan Laptop mendapat DNS Server yang benar:

C:\Users\LaptopSaya> nslookup www.lks2026.id

2. Buka Website

Buka Chrome di Laptop, akses http://www.lks2026.id.

Refresh Pertama

Ini Server WEB 01

Refresh Kedua (F5)

Ini Server WEB 02

Jika bisa dibuka dari Laptop Fisik, Setting Bridged Anda 100% SUKSES!

Troubleshooting Fisik

Dapat IP tapi tidak bisa Ping / Buka Web
Firewall: Coba matikan Windows Firewall di Laptop Client dan di PC Server (Host).
Routing: Cek di Router fw, apakah `iptables -t nat` sudah benar?
Tidak dapat IP sama sekali (Unidentified Network)
Bridging Salah: Buka Virtual Network Editor di PC Server. Pastikan VMnet0 diset ke Bridged dan pilih nama LAN Card yang benar (jangan Auto).
